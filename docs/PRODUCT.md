# PRODUCT.md - 제품 정의

## 1. 제품 개요

**Hearlink Guidam CRM v2** - 보청기 현장centercrm 시스템

보청기 판매/피팅 센터를 위한 고객 관리 + 사업 관리 시스템. 고객 한 명의 전 Life Cycle을 관리.

## 2. 주요 사용자

| 사용자 | 역할 |
|--------|------|
| **센터장/관리자** | 전체 Dashboard, 결제/문서 최종 확인 |
| **상담사/치료사** | 고객 상담, 청력검사, 일정 관리 |
| **행정담당** | 결제, 문서, 알림 관리 |

## 3. 핵심 기능 범위

### 3.1 고객 관리
- 고객 등록/수정/삭제
- 고객 기본 정보 (인적사항, 청력상태, 정부지원 유형)
- **고객 상세 허브**: 모든 정보의 중심

### 3.2 상담/검사
- 상담 기록 (센터방문/재택방문/원격)
- 청력검사 (순음역치, 어음역치)
- 청력장애 분류 (전음성/감각신경성/돌발성/소음성)

### 3.3 일정 관리
- 고객별 일정 등록
- 캘린더 뷰
- 일정 알림

### 3.4 결제 관리
- Sale (총액/입금액/미수금)
- TossPay 연동
- 결제 상태 추적

### 3.5 문서/자격
- 주민등록등본, 의사소견서 등
- 정부지원 자격 서류
- Barobill Fax 연동

### 3.6 업무일지
- 직원별 활동 기록
- 고객 방문, 전화, 피팅, 문서 준비

### 3.7 알림
- 시스템 알림
- 카카오톡 알림톡

## 4. 보청기 도메인 지식

### 4.1 고객 포지션 분류
```
자가 (SELF)         - 민간 중심, 자비 부담
타기관 (OTHER)      - 타 병원/기관介绍
히어닷컴 (HEARDOTCOM) - 히어닷컴 채널
```

### 4.2 정부지원 유형
```
장애등급소지자   - 장애 등급 1-6급
장애가망        - 장애 예상
산업재해        - 산재 인증
일반            - 정부 지원 없음
```

### 4.3 수급자 구분
```
수급자 (RECIPIENT)   - 지자체 100% 지원
차상위 (NEAR_POVERTY) - 공단 100% 지원
일반 (GENERAL)        - 공단 90% 지원
```

### 4.4 청력장애 유형
```
전음성 (CONDUCTIVE)       - 외래/중이 문제
감각신경성 (SENSORINEURAL) - 내이/청神经 문제
돌발성 (SUDDEN)           - 갑자기 발생
소음성 (NOISE_INDUCED)    - 장기적 소음 노출
```

## 5. 마일스톤

### M1: 핵심 사이클 (완료)
- 고객 CRUD + 목록
- 고객 상세 허브
- 상담/청력검사/일정/결제 등록

### M2: 나머지 탭 (진행 중)
- WorkLog
- Notification
- Document

### M3: 외부 연동 (예정)
- TossPay 결제
- Barobill Fax
- 카카오톡 알림

### M4: 고도화 (예정)
- 대시보드 통계
- 캘린더 뷰
- 엑셀 내보내기

## 6. 완료 조건

### 핵심 사이클 완료 기준
```
1. 고객 목록 조회 → OK
2. 고객 상세 조회 → OK
3. 상담 등록 → 목록 반영 → OK
4. 청력검사 등록 → 목록 반영 → OK
5. 일정 등록 → 목록 반영 → OK
6. 결제 등록 → 목록 반영 → OK
```

### 외부 연동 완료 기준
```
1. TossPay 결제 요청 → 토스 페이지 → 완료
2. Barobill Fax 전송 → 상태 확인
3. 카카오톡 알림 → 수신 확인
```
